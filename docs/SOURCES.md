# Sources Used For AtlasTrace

This project scaffold was shaped using current public material checked on May 1, 2026.

## Investigation and knowledge-graph design

- Aleph docs: investigations, datasets, search, diagrams, and cross-referencing informed the case-centric structure.
  - https://docs.aleph.occrp.org/about
  - https://docs.aleph.occrp.org/users/getting-started/homepage/
  - https://docs.aleph.occrp.org/users/search/advanced/
- OpenCTI docs: connector-style ingestion and knowledge-graph thinking informed the module and relationship model.
  - https://docs.opencti.io/latest/
  - https://docs.opencti.io/latest/usage/getting-started/

## Kerbes references

- Kerbes terms and positioning: reinforced the lawful, professional, and cross-correlation oriented framing.
  - https://www.kerbes.org/terms-of-service
  - https://kerbes.org/gov
  - https://kerbes.org/tutorial

## Extraction, metadata, and OCR

- Trafilatura docs: clean article extraction for public web pages.
  - https://trafilatura.readthedocs.io/en/latest/
  - https://trafilatura.readthedocs.io/en/latest/usage-python.html
- ExifTool docs: rich metadata extraction path for media files.
  - https://exiftool.org/exiftool_pod2.html
- Tesseract docs: optional OCR support for image-based evidence.
  - https://tesseract-ocr.github.io/tessdoc/

## Framework and future API growth

- Typer project page and docs: inspiration for future CLI ergonomics if the project outgrows argparse.
  - https://typer.tiangolo.com
  - https://pypi.org/pypi/typer/
- FastAPI docs: inspiration for the optional API layer.
  - https://fastapi.tiangolo.com/
